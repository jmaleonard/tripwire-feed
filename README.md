This is the public indicator-of-compromise (IoC) feed for
[agent-tripwire](https://github.com/jmaleonard/agent-tripwire). A daily GitHub
Actions job fetches Aikido's npm + PyPI malware lists, merges them (~130K IoCs),
and publishes them here for tripwire daemons to pull — plus a public
**[top-offenders page](https://jmaleonard.github.io/tripwire-feed/)** and a way
for anyone to report a malicious package.

The whole thing **costs nothing to run.** It rides entirely on GitHub's
free-for-public-repos services — Actions builds it, Releases host the snapshots,
Pages serves the site, Issues collect reports. There's no server and no AWS.

## What's here

- **📦 The feed** — daily snapshots + deltas of known-malicious npm/PyPI
  packages, consumed by tripwire daemons.
- **🌐 The page** — [jmaleonard.github.io/tripwire-feed](https://jmaleonard.github.io/tripwire-feed/),
  a public "top offenders" report built from the latest snapshot.
- **⚠️ Reporting** — a [report form](https://github.com/jmaleonard/tripwire-feed/issues/new?template=report-malicious-package.yml)
  (and a per-package **report** link on the page) so the community can flag
  malicious packages. Submissions are reviewed before they enter the feed.

## The feed

```
Release `feed` tag:
  snapshot-YYYY-MM-DD.json   full snapshot, 3 newest retained ← release asset (manifest points here)
  latest.json                clobbered mirror of newest        ← human convenience only

feed/v1/:
  manifest.json              index clients read first
  delta-YYYY-MM-DD.json      daily diff, last 30 retained      ← committed (small, audit trail)
```

The snapshot is date-stamped (never a reused filename), so its CDN URL is always
fresh — clobbering one `latest.json` would let Fastly serve a stale copy.

### Consume

```bash
# the index — start here; it points at the current snapshot + deltas
curl -s https://raw.githubusercontent.com/jmaleonard/tripwire-feed/main/feed/v1/manifest.json
# newest full snapshot (mirror; clients follow manifest.full.url instead)
curl -sL https://github.com/jmaleonard/tripwire-feed/releases/download/feed/latest.json
```

A tripwire daemon does this automatically on startup and every 6h
(`tripwire ioc sync` to force it): reads the manifest, downloads only the deltas
it's missing, and SHA-256-verifies every body before applying. Format and client
are documented in
[agent-tripwire `spec/docs/feed.md`](https://github.com/jmaleonard/agent-tripwire/blob/main/spec/docs/feed.md).

## The page

[`jmaleonard.github.io/tripwire-feed`](https://jmaleonard.github.io/tripwire-feed/)
is a static "top offenders" report — most recently flagged packages, biggest
campaigns, ecosystem breakdown — rebuilt from the latest snapshot on every
publish and deployed to GitHub Pages. It's generated by
`scripts/build-site.mjs`; the publish job hands it the freshly-built snapshot as
a workflow artifact, so it never depends on the release CDN.

## Reporting a malicious package

Found a malicious package that isn't on the feed? Two ways in, both free:

- **From the page** — the **⚠️ Report a malicious package** button, or the
  **report** link on any package row (it pre-fills the package name).
- **Directly** — open the
  [report form](https://github.com/jmaleonard/tripwire-feed/issues/new?template=report-malicious-package.yml).

Reports arrive as `ioc-report` issues and are **reviewed before ingestion** —
nothing is published to the feed automatically. This is detection-only awareness,
not an accusation of any maintainer.

## Publish

`.github/workflows/seed-feed.yml` runs daily at 06:00 UTC (or via
**Run workflow**). It executes the self-contained `scripts/publish-feed.mjs`
bundle and publishes to this repo with the built-in `GITHUB_TOKEN` — no secrets
to configure.

The bundle is generated from
[agent-tripwire](https://github.com/jmaleonard/agent-tripwire)
(`packages/feeds` + `scripts/publish-feed.mjs`). To update it:

```bash
# in an agent-tripwire checkout
pnpm --filter @tripwire/shared --filter @tripwire/feeds build
npx esbuild scripts/publish-feed.mjs --bundle --platform=node --format=esm \
  --outfile=/path/to/tripwire-feed/scripts/publish-feed.mjs
npx esbuild scripts/build-site.mjs   --bundle --platform=node --format=esm \
  --outfile=/path/to/tripwire-feed/scripts/build-site.mjs
```
